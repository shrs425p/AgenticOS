Capabilities.

01

Self-Evolution

Autonomously identifies missing capabilities, pip-installs libraries, and generates new tool plugins at runtime — no intervention required.

02

Rate-Limit Shield

Built-in exponential backoff absorbs all 429 errors transparently. Handled 50+ rate-limit hits during the Crucible without a single crash.

03

Tasks decompose into dependency graphs. The orchestrator resolves execution order, retries failed nodes, and adapts plans mid-flight.

Persistent Memory

Consolidated MEMORY.md for long-term recall + SQLite episodic logs retain context across sessions. Rolling summarisation keeps token budgets lean automatically.

Architecture.

Orchestrator

Central runtime loop. Receives tasks, resolves DAG execution order, dispatches to tools, and coordinates memory reads/writes per step.

Tool Registry

Auto-discovers 350+ modular tools from the tools/ directory. Plugins hot-load at runtime — the agent can register new tools mid-session.

Memory Layer

Dual-store: Consolidated MEMORY.md for semantic long-term context retention, SQLite for structured episodic session logs. Rolling summarisation keeps token usage lean.

Self-Heal Engine

On any tool failure, retries up to 7 attempts with backoff, auto-adjusts parameters, and logs every recovery attempt to the persistent audit trail.

Security Guardrails

PathGuard enforces zone-based filesystem access. HITM prompts human confirmation for sensitive operations. Command validation blocks dangerous shell patterns.

Zone-Based
Security.

Green Zone

workspace/

Full autonomous access. The agent operates freely — creates files, runs tasks, stores artifacts. No confirmation required.

Yellow Zone

%USERPROFILE%/Documents, Desktop…

Human-in-the-Middle (HITM) mode. Every write or delete operation requires explicit user confirmation before execution.

Red Zone

C:\Windows, /System, /Library, /etc…

Hard-blocked. PathGuard enforces absolute platform-level path guardrails. No operation in this zone executes under any circumstance.
