Architecture.

Orchestrator

Central runtime loop. Receives tasks, resolves DAG execution order, dispatches to ops, and coordinates memory reads/writes per step.

Tool Registry

Auto-discovers 250 modular ops from the ops/ directory. Plugins hot-load at runtime — the agent can register new ops mid-session.

Memory Layer

Dual-store: Consolidated MEMORY.md for semantic long-term context retention, SQLite for structured episodic session logs. Rolling summarisation keeps token usage lean.

Self-Heal Engine

On any tool failure, retries up to 7 attempts with backoff, auto-adjusts parameters, and logs every recovery attempt to the persistent audit trail.

Security Guardrails

AST-based command tokenization with shlex parsing intercepts shell chaining, obfuscation, and encoded payloads. PathGuard enforces zone-based filesystem access. Line-by-line script validation scans every instruction before execution.

Zero-Trust
Security.

Every command is structurally parsed before execution. Every file path is boundary-checked. Every script is read line-by-line. No shortcuts. No bypasses.

AST Parsing

Command Tokenizer

shlex-based structural tokenization deconstructs nested quotes, variable expansions, and argument splits — catching obfuscation that regex validators miss.

Chain Detection

Operator Interception

Detects and blocks shell chaining operators (&&, ;, ||, |) and obfuscated commands before they reach the subprocess layer.

Script Scanning

Line-by-Line Validation

Reads .sh, .bash, .ps1, .bat, and .cmd files, filters comments, merges line continuations, and validates every instruction against safety rules.

PowerShell Defense

Encoded Command Audit

Detects -enc, -e, -en parameter prefix matching and decodes base64-encoded PowerShell payloads to audit the underlying script before execution.

PathGuard Zones

Green Zone

workspace/

Workspace isolation. The agent operates freely inside the workspace root. Write or delete operations outside the workspace require human confirmation (HITM).

Yellow Zone

System-wide autonomy

PathGuard is active but outside-workspace modifications are allowed autonomously. Explicitly blocked paths (Red Zone) are still forbidden.

Red Zone

PathGuard Disabled

The kernel guardrail engine is disabled entirely. The agent has unrestricted filesystem access and can read or write anywhere on the system.

Blue Zone

Read-Only / Audit

All write and delete operations are blocked system-wide. The agent has read-only access, making it ideal for non-destructive code review and security audits.
