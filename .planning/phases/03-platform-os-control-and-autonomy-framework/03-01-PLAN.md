<threat_model>
## Trust Boundaries
- Subprocess execution boundary for running Applescript `osascript` on macOS, and `grim`/`scrot` screenshot utilities on Linux.
- Windows API security boundary when querying process elevation tokens or dispatching WScript.Shell COM calls.

## STRIDE Register
| ID | Threat | Component | Mitigation |
|----|--------|-----------|------------|
| T-03-01 | Elevation of Privilege | windows_ui.py | Detect if the targeted process is UAC-elevated; if so, raise AgentError advising the user to run the agent in admin mode rather than silently failing to control it. |
| T-03-02 | Tampering / Command Injection | macos_ui.py | Subprocess calls to `osascript` must escape user strings properly or construct Applescripts dynamically without format-injection vulnerabilities. |
| T-03-03 | Denial of Service / Hanging | macos_ui.py / linux_desktop.py | Impose 5-second timeouts on all GUI subprocess calls to prevent blocking the agent execution loop indefinitely if a GUI utility hangs. |
</threat_model>

## Artifacts this phase produces
- `tools/platform/__init__.py`: Cross-platform dispatcher routing GUI operations to appropriate OS backends.
- `tools/platform/windows_ui.py`: Native Windows GUI automation implementation using pywin32.
- `tools/platform/macos_ui.py`: macOS GUI automation using osascript (AppleScript).
- `tools/platform/linux_desktop.py`: Linux session detection and screenshots wrapper (grim/scrot).
- `tests/test_platform_ui.py`: Unit tests covering all platforms with mocks.
