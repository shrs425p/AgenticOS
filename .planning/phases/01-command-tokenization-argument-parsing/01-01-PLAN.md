    3. Add private helper method `_detect_obfuscation(token)` to flag when a token has internal quotes and the cleaned token consists entirely of alphanumeric characters, underscores, or hyphens (using `re.match(r"^[a-zA-Z0-9_-]+$", cleaned)`).

    6. Return a block reason if `_detect_obfuscation(token)` is true for any token: `Command blocked by safety rules: command obfuscation detected (internal quotes in token: {token})`.
    7. Perform structural exact-match validations against `rules` dict (e.g. checking if first token cleaned is `sc` or `shutdown`, or if first token is `net` and second is `stop`/`start`) rather than the old substring searches.
