  <action>
    Implement `SafetyMixin._get_redirection_targets(self, command: str) -> list[str]`:
      - Regex match redirection operators (`>` and `>>`) followed by filenames (handling single, double, and unquoted cases).
      - Regex match pipeline tools (`tee`, `tee-object`, `out-file`, `set-content`, `add-content`) followed by target path parameters.
    Integrate into `SafetyMixin._blocked_command_reason`:
      - For each extracted redirection target, verify if the extension matches `.py`, `.sh`, `.ps1`, `.bat`, `.cmd`, `.vbs`, or `.js`.
      - If it does, run `self.guard.check_path(str(target), operation="write")`.
      - If `check_path` returns `False`, check if the message is `"HITM_REQUIRED"`. If so, invoke `self.guard.ask_human(str(target), "write_redirect")`. If that returns `False` (rejected), block command execution and return a security policy message. If not "HITM_REQUIRED", block it outright.
  </action>
  <verify>Run `pytest tests/test_safety.py -k test_code_gen_redirect`</verify>
  <done>Redirection targets are parsed and validation blocks unauthorized script creation outside the workspace.</done>
</task>
