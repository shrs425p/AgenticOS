## 4. E2E Flow & Integration Audit

### **E2E Command Execution Flow**
The integration checker traced the full path of command execution:
1. **Tool Invocation**: Orchestrator triggers `run_command` or other tools via `ToolRegistry`.
2. **Path Verification**: Gated by `PathGuard` against configuration and sensitive path violations.
3. **Safety Interception**: `RunnerMixin._run` passes the command to `SafetyMixin._blocked_command_reason`.
4. **Validation Logic**:
   - Shell chaining check (detects `;`, `&&`, etc.).
   - Tokenization & Obfuscation check (removes quotes/escapes, verifies variable expansions).
   - Recurse command wrapper check (handles nested `cmd /c`, `powershell -c`, `bash -c`).
   - PowerShell flag prefix & Base64 UTF-16LE decoding recursively validates obfuscated parameters.
5. **Security Auditing**: Validation failures write structured error entries to SQLite and `errors.jsonl` under `where="security_validation"`.

All connections between safety validation (`SafetyMixin`) and execution layers (`RunnerMixin` & `AgentRuntime`) are fully verified.
