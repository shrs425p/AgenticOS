## Trust Boundaries
- Subprocess execution feedback and logs parsing to analyze stdout/stderr messages.
- Task completion verification evaluating environmental status indicators.

## STRIDE Register
| ID | Threat | Component | Mitigation |
|----|--------|-----------|------------|
| T-03-06 | Tampering / Error Evasion | retry_classifier.py | Do not let transient retries run infinitely (cap transient error retries strictly at 3, and ambiguous ones at 2) to prevent execution starvation. |
| T-03-07 | Denial of Service | stall_monitor.py | Prevent tool execution hangs by alerting early on categorized command categories (file operations, network, package installations). |
</threat_model>

## Artifacts this phase produces
- `core/retry_classifier.py`: Classification logic for transient vs permanent errors using hybrid exit codes and regex.
- `core/stall_monitor.py`: Execution time tracker raising warnings and listing optimized alternatives.
- `core/success_criteria.py`: Goal parser extracting explicit requirements and checking them at completion.
- `docs/deployment.md`: Service playbooks for Docker Compose, Windows Service (NSSM), and Kubernetes.
- `tests/test_retry_classifier.py`: Suite checking error classifications.
- `tests/test_stall_monitor.py`: Suite checking category thresholds and warning output.
- `tests/test_success_criteria.py`: Suite verifying extraction patterns.

  <action>
    Create `core/retry_classifier.py`.
    - Define `RetryDecision` dataclass: `action` ("retry" | "abandon" | "escalate"), `reason`, `max_retries`.
    - Define `RetryClassifier` class:
      - Lists of transient regex patterns (`timeout`, `network`, `connection reset`, etc.) and permanent patterns (`permission denied`, `not found`, `syntax error`, etc.).
      - `classify(error_msg: str, exit_code: int | None = None) -> RetryDecision`: exit code is primary check (1, 2, 127, 128, 130 represent permanent exit codes). Fallback to message pattern checking. Returns RetryDecision.
  </action>
