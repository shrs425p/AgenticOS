# Contributing to AgenticOS

First off, thank you for considering contributing to AgenticOS! It's people like you that make AgenticOS such a great tool.

## [RULES] Code of Conduct
By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## [FLOW] How Can I Contribute?

### Reporting Bugs
*   **Check the Issues**: Before reporting a bug, please search existing issues to see if it has already been reported.
*   **Use the Template**: If you find a new bug, please use the **Bug Report** template to provide as much detail as possible.

### Suggesting Enhancements
*   **Open an Issue**: Use the **Feature Request** template to explain your idea and how it benefits the project.

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure the test suite passes.
4.  Make sure your code follows the existing style (run `ruff check .`).
5.  Issue that pull request!

## [STYLE] Style Guide
*   **Python**: We follow PEP 8. Please use `ruff` for linting and formatting.
*   **Documentation**: Ensure any new features are documented in the `docs/` directory.

## [HELP] Questions?
If you have questions, please open a **Discussion** or an issue labeled `question`.
